1.3 Gate 시스템 (단계별 관문)

각 단계(Phase)를 넘어가기 전에 반드시 통과해야 하는 의사결정 관문(Gate)을 둔다. Gate를 통과하지 못하면 다음 단계로 진행할 수 없으며, 이를 통해 감(感)이 아닌 데이터 기반의 의사결정을 구조화한다.

Table 3

| Gate | 명칭 | 위치 | 핵심 판단 |
| --- | --- | --- | --- |
| G0 | 사업 진입 검토 | 사업기획 완료 후 | 이 토지/사업에 진입할 가치가 있는가? |
| G1 | 토지 매입 의사결정 | 실사 완료 후 | 이 토지를 매입해도 안전한가? |
| G2 | 사업 확정 | 사업타당성 분석 후 | 사업수지가 목표 수익률을 충족하는가? |
| G3 | PF 착수 승인 | 인허가 진행 중/완료 후 | PF 조달을 위한 조건이 갖춰졌는가? |
| G4 | 분양 개시 승인 | 분양 준비 완료 후 | 분양 시점과 조건이 적절한가? |
| G5 | 준공 정산 확인 | 준공 후 | 모든 채무가 정리되고 수익이 확정되었는가? |

| 부동산 브로커/중개사 네트워크 | 비공개 매물 접근 가능, 시장 분위기 파악 | 브로커 수수료 구조 확인 필수, 허위 매물 주의 |

| 직접 발굴 (현장답사, 토지대장 분석) | 남들이 모르는 기회 포착, 협상력 우위 | 시간·인력 소요 큼 |

6-3. [v1.0] 시공사 부도 시 유치권 방어 플랜
